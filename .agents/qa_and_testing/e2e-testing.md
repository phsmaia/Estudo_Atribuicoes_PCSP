# Testes Fim a Fim (E2E)

Esta regra define como testamos o sistema pelo ponto de vista do usuário final, garantindo que as peças individuais funcionem em conjunto.

## Testando Fluxos Críticos
- Utilize ferramentas modernas de E2E (como Playwright ou Selenium) para simular interações reais na interface.
- Os testes devem cobrir os fluxos principais (ex: login, navegação, envio de formulários).

## Integração Orgânica
- **Com o Visual (`frontend-design.md`):** Os testes E2E garantem que o design premium e as interações continuem funcionando sem quebrar silenciosamente.
- **Com a Acessibilidade (`ui-ux-accessibility.md`):** Ferramentas de E2E devem ser configuradas para auditar acessibilidade de forma automatizada durante a navegação.
