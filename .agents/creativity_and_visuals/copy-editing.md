# Copy Editing

Esta regra governa a linguagem utilizada nas interfaces de usuário (UI), tooltips, mensagens de erro voltadas para o usuário e landing pages.

## Tom e Concisão
- A linguagem deve ser clara, concisa, persuasiva e livre de jargões técnicos excessivos ao falar com o usuário final.
- Mensagens de erro devem ser amigáveis e oferecer um caminho de solução (ao invés de "Erro 500: Falha no DB", use "Tivemos um problema interno, por favor tente novamente em instantes").

## Sinergia e Conflitos
- **Diferenciação:** Diferente do `technical-writing.md` (que é voltado para desenvolvedores e foca em precisão técnica rigorosa), o `copy-editing` é focado em UX (Experiência do Usuário). Eles não conflitam, pois atuam em escopos distintos (Copy = Interface; Tech Writing = README e Docstrings).
